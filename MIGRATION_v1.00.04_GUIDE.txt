## Benefits of New Architecture

1. **Easy to Extend**: Adding Part-Time/Full-Time Jobs now requires only:
   - Add job data to config/
   - Add logic to domains/occupation.js
   - Add UI to components/panels/OccupationPanel.js

2. **Easy to Debug**: Each feature isolated in its own file

3. **Easy to Test**: Pure functions can be tested independently

4. **Easy to Read**: ~100 line files instead of 1000+ lines

5. **Performance**: No change - same React Native optimization

## Adding New Features (Example: Part-Time Jobs)

### Step 1: Add job data to config/
Create: src/config/jobs.js
```javascript
export const partTimeJobs = [
  { id: 'barista', name: 'Barista', pay: [15, 25], hours: 20, stressGain: 5 },
  { id: 'retail', name: 'Retail Worker', pay: [12, 20], hours: 20, stressGain: 7 },
];
```

### Step 2: Add logic to domain
Update: src/domains/occupation.js
```javascript
export function handlePartTimeJob(life, jobId) {
  const updated = deepClone(life);
  const job = partTimeJobs.find(j => j.id === jobId);
  // ... job logic
  return updated;
}
```

### Step 3: Add UI to panel
Update: src/components/panels/OccupationPanel.js
```javascript
<TouchableOpacity onPress={() => onPartTimeJob('barista')}>
  <Text>Work as Barista</Text>
</TouchableOpacity>
```

### Step 4: Wire to App.js
Update: src/App.js
```javascript
import { handlePartTimeJob } from './domains/occupation';

function handlePartTimeJobClick(jobId) {
  setLife(handlePartTimeJob(life, jobId));
}
```

## Architecture Principles

1. **App.js = Orchestration Only**: Only handles state and routing
2. **Domains = Business Logic**: Pure functions (life, params) => updatedLife
3. **Components = UI Only**: Pure presentation, emit events to App.js
4. **Config = Static Data**: Name lists, job definitions, etc.
5. **Core = Foundational Logic**: Life object management, NPC generation
6. **Utils = Helper Functions**: Random utilities, formatters
